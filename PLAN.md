## GitHub Pages 向け静的フロントエンド実装計画

### Summary
GitHub Pages で `/docs/index.html` から動く、React + Vite + TypeScript の静的ダッシュボードを作る。  
Pages 版はブラウザのみで完結させ、既存の保存済み要約・埋め込み・市場データを軽量派生 JSON として再利用する。  
Gemini は「常時接続」前提とし、ページ初回表示時に API キー入力モーダルを開き、`localStorage` 保存のみで扱う。ただしブラウザ実行である以上、キーは秘匿できないことを UI 上で明記する。  
株価予想表示は、既存データと整合する形で「類似日の当日騰落率」を参考値として表示する。

### Key Changes
- フロントエンド構成
  - `frontend/` に Vite + React + TypeScript のソースを置き、ビルド出力先を `docs/` に固定する。
  - GitHub Pages では `docs/index.html` と静的アセットだけで動作する構成にする。
  - ルーティングは不要なため SPA 1 ページ構成にし、HashRouter も使わない。

- 公開用データ構成
  - `backend/data/...` の既存生成物をそのまま UI から読むのではなく、Pages 用の軽量派生 JSON を作る前提にする。
  - 含める内容は 6 営業日分の以下に限定する。
    - 日付
    - ニュース件数
    - サンプル見出し
    - 日次サマリー本文
    - 3072 次元埋め込みベクトル
    - 類似日比較に必要な市場データ
    - 表示用の主要ジャンル・国・企業
  - `news_current.csv` などの大きい原データは Pages UI の直接入力には使わない。
  - 静的データは `docs/assets/data/*.json` のような形で配信し、初回ロードで取得する。

- 秘匿情報と公開チェック
  - 実キーのコミットは行わない。
  - リポジトリ内の確認対象は `.env`、ソース、ビルド成果物、`docs/` 配下、生成 JSON。
  - 現時点の探索では実キーは見つかっておらず、ヒットしたのは公式ドキュメントの `GEMINI_API_KEY` 例と既存 Python の環境変数参照のみ、という前提で進める。
  - UI に以下を明記する。
    - API キーはブラウザで使用されるため秘匿できない
    - `localStorage` に保存される
    - 共用端末では保存しないこと
    - デモ用キーや利用制限付きキーを推奨すること

- ダッシュボード UI
  - 1 ページ内に工程順でカードを並べる。
    - ヘッダー / 概要
    - API 管理
    - ニュース入力
    - 日次サマリー
    - ベクトル
    - 類似度ランキング
    - 予想株価
    - 処理ログ / 現在ステップ
  - 各カードは `概要 / 詳細` のタブ切替を持たせる。
  - 画面上部にパイプライン進行表示を置き、現在の計算段階をハイライトする。
  - レスポンシブはモバイル 1 カラム、タブレット以上で 2 カラム、デスクトップで主要カードを横断配置する。

- 入力体験
  - 「自由入力中心」にする。
  - ニュース入力欄では任意テキスト貼り付けを主導線にしつつ、保存済み 6 営業日のサンプル読み込みボタンも置く。
  - サマリー欄とベクトル欄は手入力も可能にし、各段階で「既存データを使う / Gemini で再計算する」を切り替えられるようにする。
  - API 使用量温存のため、保存済み日付を選んだ場合は既存要約・既存埋め込みを優先使用し、再計算は明示操作時のみ行う。

- クライアント側ロジック
  - Gemini 要約は `gemini-3-flash-preview`、埋め込みは `gemini-embedding-001` をブラウザから直接呼ぶ。
  - ベクトル類似度はクライアント側でコサイン類似度を計算する。
  - 3072 次元の既存ベクトル群を読み込み、入力ベクトルとの上位 N 件ランキングを返す。
  - 予想株価は、最上位類似日または上位数件の加重平均を使って、`^N225` と `1306.T` の当日騰落率参考値を表示する。
  - 予想は「参考表示」であり投資判断用ではないことを明記する。

- 主要コンポーネント / 公開インターフェース
  - `ApiKeyModal`
    - API キー入力、保存、削除、注意事項表示
  - `PipelineStatus`
    - `idle | ready | summarizing | embedding | ranking | predicted | error`
  - `NewsPanel`
    - 自由入力、サンプル読み込み、文字数表示
  - `SummaryPanel`
    - 既存要約表示、Gemini 再要約、手動編集
  - `VectorPanel`
    - 既存ベクトル表示、埋め込み実行、ベクトル長表示、簡易可視化
  - `SimilarityRankingPanel`
    - 上位類似日一覧、スコア、主要トピック、参照要約
  - `PredictionPanel`
    - 予想騰落率、参照日、根拠の簡易説明
  - データ型は少なくとも以下を持つ。
    - `DailyRecord`
    - `MarketRecord`
    - `SimilarityResult`
    - `PipelineState`
    - `ApiKeyState`

### Test Plan
- 静的配信
  - `vite build` 後に `docs/index.html` から相対パスで正常表示される。
  - GitHub Pages のサブパス配信でもアセット読込に失敗しない。

- API キー管理
  - 初回表示でモーダルが開く。
  - 入力したキーが `localStorage` に保存され、再訪時に再利用される。
  - キー削除操作で即時に消える。
  - キー未設定時は Gemini 実行ボタンが無効になる。

- 既存データ利用
  - 保存済み 6 営業日を切り替えて、ニュース、要約、3072 次元ベクトル、ランキング、予想表示が一貫して更新される。
  - サンプル選択時に Gemini API を呼ばずに表示できる。

- Gemini 呼び出し
  - 自由入力ニュースから要約生成ができる。
  - 要約文から埋め込みが取得できる。
  - エラー時に画面上で失敗段階が分かる。

- 類似検索 / 予想表示
  - 同一日サンプルでは自己類似が最上位になる。
  - ランキング上位がスコア順に並ぶ。
  - 予想欄に `^N225` と `1306.T` の当日騰落率参考値が表示される。

- セキュリティ / 公開前確認
  - `docs/` 出力物に実キーが含まれない。
  - ソース、静的 JSON、HTML、JS に実キー文字列が混入していない。
  - 注意文言が UI 上で確認できる。

### Assumptions
- Pages 版はデモ用途であり、API キーのブラウザ露出リスクを許容する。
- 保存済みデータは 2025-06-23 から 2025-06-30 の 6 営業日分を初期データセットとする。
- 株価予想は当日騰落率の参考表示に限定し、翌日以降のリターン分析は Pages 版の対象外とする。
- 本番の FastAPI 連携は今回含めず、将来差し替えやすいようフロント側のデータ取得層を分離しておく。
